# Project Structure

## Directory Layout

```
03-pizza-menu/
├── public/                    # Static files served by the app
│   ├── index.html            # Main HTML template
│   ├── favicon.ico           # Browser tab icon
│   ├── manifest.json         # PWA manifest
│   ├── robots.txt            # SEO robots config
│   └── pizzas/               # Pizza images
│       ├── focaccia.jpg
│       ├── margherita.jpg
│       ├── spinaci.jpg
│       ├── funghi.jpg
│       ├── salamino.jpg
│       └── prosciutto.jpg
│
├── src/                       # React source code
│   ├── index.js              # Main app and React components
│   ├── index.css             # Global and component styles
│   └── data.js               # Data (pizzas, users, orders)
│
├── node_modules/             # Project dependencies (auto-generated)
│
├── build/                     # Production build (generated by npm run build)
│
├── package.json              # Project metadata and dependencies
├── package-lock.json         # Locked dependency versions
│
├── .gitignore                # Files to ignore in git
├── .env.example              # Environment variables template
│
├── README.md                 # Project documentation
├── CONTRIBUTING.md           # Contributing guidelines
├── DEPLOYMENT.md             # Deployment instructions
├── LICENSE                   # MIT License
└── .git/                      # Git version control (after init)
```

## File Descriptions

### Root Files

| File | Purpose |
|------|---------|
| `package.json` | Project metadata, dependencies, and build scripts |
| `package-lock.json` | Locked versions of dependencies |
| `.gitignore` | Files to exclude from git repository |
| `.env.example` | Template for environment variables |
| `README.md` | Main project documentation |
| `CONTRIBUTING.md` | Guidelines for contributors |
| `DEPLOYMENT.md` | Deployment instructions |
| `LICENSE` | MIT License text |

### Public Directory

| File | Purpose |
|------|---------|
| `index.html` | HTML template (only file directly accessed) |
| `favicon.ico` | Browser tab icon |
| `manifest.json` | PWA configuration |
| `robots.txt` | Search engine directives |
| `pizzas/` | Pizza product images |

### Source Directory

| File | Purpose |
|------|---------|
| `index.js` | Main React app and all components |
| `index.css` | All styling for the application |
| `data.js` | Pizza menu, user database, order statuses |

## Component Architecture

### React Components (in `src/index.js`)

```
App
├── Header
├── LoginPage
├── SignUpPage
├── Menu
│   └── Pizza (multiple)
├── Cart
├── OrderHistory

```

## Data Flow

```
App State:
├── currentUser: { email, name }
├── cart: [{ pizza objects }]
├── orders: [{ order objects with status }]
├── database: { email → user data }
└── showOrderHistory: boolean
```

## Development vs Production

### Development (`npm start`)
- Hot reload enabled
- Source maps for debugging
- No minification
- All files readable

### Production (`npm run build`)
- Minified code
- Optimized bundle size
- Cache-busting with hashes
- Ready for deployment

## Adding New Features

### To Add a New Pizza:
1. Add to `pizzaData` array in `src/data.js`
2. Add image to `public/pizzas/`

### To Add a New User:
1. Add to `customerDatabase` in `src/data.js`
2. Users can also sign up dynamically

### To Add New Routes:
1. Create new component in `src/index.js`
2. Add conditional rendering in App component
3. Add navigation buttons in Header

## Environment Variables

Create `.env` file:
```
REACT_APP_API_URL=https://api.example.com
REACT_APP_ENVIRONMENT=production
```

Access in code:
```javascript
const apiUrl = process.env.REACT_APP_API_URL;
```
